Clarte: A set of flexible tools for common fMRI tasks.¶

At the center of the toolbox is the fmri_data object, which is loosely based on the fmri_data object from the Canlab toolbox (https://github.com/canlab). Its primary role is to load, mask, resample, and reshape fMRI data, but many functions are designed to interact with it.

Two different tools for searchlight analyses (seachlightSphere and searchlightKnn) are available. They are flexible, support any analysis or scoring method, and should be intuitive to use.

Non-Standard/Non-Python Dependencies:¶

  • nibabel, nilearn, seaborn, pysurfer, tqdm, matlab_wrapper, nipype
  • SPM, FSL, BROCCOLI, ANTs, freesurfer

To demonstrate the use of clarte, we will run a searchlight analysis using single-trial t-maps derived from the least-squares separate procedure (Mumford et al., 2012; NeuroImage). First we import modules, and organize the data.

Now we can load the data using the fmri_data object. It requires at least two inputs, the first is a list of filenames or nibabel image objects, and the second is a mask.

In this example, we will automatically calculate a background mask, but many options are possible (e.g., a path to a specific mask). One of the nice features of the toolbox is that masks are automatically resampled to the space of the functional images.

In [4]:
dat = cl.fmri_data(df['imagePath'].values,'background')
print('dat.dat.shape:',dat.dat.shape)

dat.dat.shape: (240, 24001)

We have now loaded the 240 images, and reshaped this 4D info into a 2d matrix (240 images * 24001 voxels), a convenient form for many different analyses.

If we had specified the optional 'fwhm' parameter (e.g., "dat = cl.fmri_data(df['imagePath'].values, 'background', fwhm=6"), each image would have been smoothed with the specified isovoxel kernel.

Let's take a look at the first image in the dat.dat matrix. We can easily transform this from a 1d vector back into a 3d nibabel image using the dat.unmasker method.

(Note that because we are in subject-space, the image won't align with the background image provided by the nilearn plot_glass_brain function)

To run a searchlight analyses, we add 2 more subfields to the dat object:¶

  • dat.pipeline: a function which defines the operation performed within each sphere
  • dat.y: the y vector sent to dat.pipeline.
In [8]:
# let's use sklearn to define our pipeline function:
from sklearn.cross_validation import LeaveOneLabelOut,cross_val_score
from sklearn.svm import LinearSVC

# define our pipeline a leave-one-run out, svm classification:
cv  = LeaveOneLabelOut(labels=dat.sessions)
clf = LinearSVC(C=.01)

# the pipeline function:
def pipeline(X,y):
    return cross_val_score(clf,X,y=y,scoring='accuracy',cv=cv).mean()

dat.pipeline = pipeline
dat.y  = df['stimulus'].values

# let's run the searchlight with sphere radius=5mm:
im = cl.searchlightSphere(dat,5)
